# 候选人画像（Profile Snapshot）设计指南

## 设计理念

候选人画像是简历的"封面"，占据简历顶部约 1/4 空间。面试官 7 秒内看到的全部就是这个区域。它的目标是让面试官在极短时间内形成对候选人的精准认知。

**核心思想**：把候选人当作一个"产品"来呈现——先给出产品概览（画像），再展开产品详情（后续章节）。

## 画像结构

候选人画像由 4 个层次组成，从上到下：

```
┌─────────────────────────────────────────────────────────────┐
│ 第1层：基础信息行                                            │
│ 姓名 | 年龄 | 毕业院校 | 工作年限 | 当前职位 | 联系方式      │
├─────────────────────────────────────────────────────────────┤
│ 第2层：行业 & 履历概览                                       │
│ 擅长行业：金融 · 电商                                        │
│ 公司履历：阿里巴巴(3年) → 美团(2年) → 字节跳动(至今)         │
├─────────────────────────────────────────────────────────────┤
│ 第3层：核心技能标签                                          │
│ Java · Spring Cloud · Kubernetes · 微服务 · 分布式 · DDD     │
├─────────────────────────────────────────────────────────────┤
│ 第4层：一句话定位 ★                                          │
│「10年Java架构师，主导过3个千万级用户系统从0到1，             │
│  擅长高并发架构设计与团队技术体系建设」                       │
└─────────────────────────────────────────────────────────────┘
```

## 各层详细设计

### 第1层：基础信息行

**目的**：让面试官 1 秒内获取候选人基本画像。

**格式**：
```markdown
# 张三

> 32岁 · 硕士/浙江大学计算机科学 · 10年经验 · 资深Java架构师
> 📱 138-xxxx-xxxx · 📧 zhangsan@email.com · 🔗 github.com/zhangsan
```

**要点**：
- 姓名用一级标题 `#`，醒目
- 基本信息用引用块 `>` 聚合为一行
- 联系方式紧跟其后（手机、邮箱、GitHub/博客）
- 年龄可写可不写（根据个人偏好），但工作年限必须写
- 院校如果知名，带上学校名；否则只写学历

### 第2层：行业 & 履历概览

**目的**：让面试官快速判断"这个人的背景跟我们匹配吗"。

**格式**：
```markdown
**擅长行业**：金融科技 · 电子商务 · 企业级SaaS

**公司履历**：阿里巴巴集团(3年) → 美团(2年) → 字节跳动(4年，至今)
```

**要点**：
- 行业用中文，2-3个即可
- 公司履历用箭头 `→` 连接，形成时间线
- 括号内标注在职年限
- 知名公司是加分项，醒目展示
- 如果公司不知名，附加一句公司简介

### 第3层：核心技能标签

**目的**：让面试官快速了解候选人的技术定位和核心能力。

**格式**：
```markdown
**核心技能**：`Java` `Spring Cloud` `Kubernetes` `微服务架构` `高并发` `分布式系统` `DDD` `团队管理`
```

**选择标准**：
- 只选 **6-8 个**最强的、最能代表你的技能
- 这不是技能列表，是"技能名片"
- 优先选择：目标岗位要求的 + 你真正精通的 + 有差异化的
- 用行内代码 `` ` `` 包裹，视觉上形成标签效果

**错误示范**：
```
❌ 列了20个技能，什么都会 = 什么都不精
❌ 只列编程语言，没有架构能力和软技能
❌ 列了太多"了解"级别的技能
```

### 第4层：一句话定位 ★

**这是整份简历最重要的一句话。**

**目的**：用一句话回答面试官心中的问题——"这个人能给我们带来什么？"

**公式**：
```
「{工作年限}{核心角色}，{最突出成就/数据}，{差异化优势/擅长领域}」
```

**优秀案例**：

```
「10年Java架构师，主导过3个千万级用户系统从0到1，擅长高并发架构设计与团队技术体系建设」

「全栈工程师 × AI技术布道者，GitHub 5000+ Stars，在金融和电商领域拥有深度技术积累」

「8年后端开发，深耕支付与交易领域，设计的交易引擎日处理金额超50亿」

「前阿里P7，6年分布式系统经验，主导完成2次核心系统重构，团队效能提升200%」
```

**写作要点**：
- 控制在 1-2 行以内（30-60字）
- 必须包含至少1个量化数据
- 突出"你有而别人没有"的差异化
- 避免空洞的形容词（"优秀的"、"丰富的"）
- 用 `「」` 包裹，视觉上强调这是核心定位

## 常见误区

| 误区 | 正确做法 |
|------|----------|
| 画像区域信息过多，像一篇文章 | 保持简洁，每层1-2行 |
| 没有一句话定位 | 这是必须的，是简历灵魂 |
| 核心技能列了15个以上 | 只选6-8个最强的 |
| 一句话定位没有数据 | 至少包含1个量化成就 |
| 公司履历没有时间 | 标注年限，形成时间线 |
| 基础信息分散在各处 | 聚合到顶部一行内 |
