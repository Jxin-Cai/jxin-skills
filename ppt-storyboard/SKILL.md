# PPT 叙事分镜设计器

将文章内容转化为面向特定受众的演示分镜，生成图片并合成 PDF。

## 核心理念

不同受众关注不同维度。给管理层看的和给技术团队看的，即使是同一篇文章，叙述重点、信息密度、视觉风格都应不同。本技能先设计叙事策略，再落地为分镜，确保每页PPT都为目标受众服务。

## 执行流程

### Step 1: 获取输入

需要两个关键输入：

1. **文章内容**（必需）：Markdown 文件路径、URL、或直接粘贴的文本
2. **叙事对象**（必需）：演示的目标受众

如果用户没有指定叙事对象，**必须先澄清**。用 AskUserQuestion 询问：

```
这份演示是给谁看的？不同受众会影响叙事策略和信息密度。

常见受众类型：
- 管理层 / C-Level：关注战略价值、ROI、风险
- 投资方：关注商业模式、市场规模、增长潜力
- 技术团队：关注架构、实现细节、技术选型
- 客户/合作伙伴：关注价值主张、案例、落地效果
- 通用受众：平衡各方面信息
```

### Step 2: 分析内容，设计叙事策略

根据文章内容和叙事对象，制定叙事策略：

**叙事策略维度**：

| 受众类型 | 信息密度 | 叙事重心 | 视觉风格 |
|---------|---------|---------|---------|
| 管理层 | 低（每页1个核心观点） | 价值、影响、决策建议 | 简洁大气、数据驱动 |
| 投资方 | 中（数据+故事） | 市场、增长、竞争优势 | 专业商务、图表丰富 |
| 技术团队 | 高（可包含细节） | 架构、流程、技术决策 | 清晰结构、流程图多 |
| 客户 | 低-中（聚焦价值） | 痛点→方案→效果 | 案例驱动、视觉冲击 |
| 通用受众 | 中 | 故事线清晰 | 平衡图文 |

### Step 3: 设计分镜

基于叙事策略，将文章内容拆分为分镜序列。每个分镜对应PPT的一页。

**分镜结构**：

```
分镜 1: [封面]
├── 标题：演示主题
├── 副标题：场景描述或日期
└── 图片提示词：封面视觉设计描述

分镜 2: [章节引导 / 内容页]
├── 标题：本页核心观点
├── 要点：3-5 个关键信息
├── 叙事备注：这页要传达什么、为什么放在这里
└── 图片提示词：本页视觉设计描述

...

分镜 N: [结束页]
├── 标题：总结/Call to Action
└── 图片提示词：结束页视觉设计
```

**设计分镜时的原则**：
- 每个分镜的图片提示词要**自包含**，包含完整的视觉描述、配色、布局要求
- 提示词用英文描述视觉构图，中文只出现在需要显示的标题/文本中
- 所有分镜共享统一的视觉风格（配色、字体、Logo位置等）
- 分镜数量建议：管理层 8-12页，技术团队 15-20页，其他 10-15页

### Step 4: 展示分镜计划，等待用户确认

将设计好的分镜以清晰的列表展示给用户：

```
📋 叙事分镜计划

🎯 叙事对象：管理层
📄 文章来源：xxx.md
📊 共 10 页分镜

分镜序列：
1. 🎬 封面 - AI驱动的研发效率提升
2. 📊 背景 - 当前研发效率的挑战
3. 💡 核心观点 - AI赋能的三个关键领域
4. 📈 数据 - 效率提升量化指标
5. 🔧 方案 - 落地路径
...
10. 🎯 结束 - 下一步行动建议

是否满意这个分镜计划？
- 满意 → 接下来选择视觉样式
- 需要调整 → 请告诉我要修改哪些分镜
```

**重要**：此步骤必须等待用户明确确认，不可自动跳过。

### Step 5: 选择视觉样式

用户确认分镜后，引导用户选择或描述样式。用 AskUserQuestion 提供选项：

```
请选择 PPT 的视觉样式：

1. ThoughtWorks 企业风格 - 深青+粉红配色，专业咨询感
2. 专业商务 - 蓝灰色系，稳重大气
3. 现代简约 - 黑白为主，强调留白
4. 自定义 - 描述你想要的样式
```

如果用户选择自定义样式，根据用户描述生成样式配置。

选定样式后，将样式要求融入每个分镜的图片提示词中，确保全局风格统一。

### Step 6: 生成图片

用户确认样式后，开始逐个分镜调用 `gemini-image-gen` 技能生成图片。

**关键执行步骤**：

1. 创建输出目录：`<用户指定路径或当前目录>/ppt-<名称>/`
2. 在 `prompts/` 子目录保存每个分镜的提示词文件
3. 在 `images/` 子目录存放生成的图片

**频率控制**（由 gemini-image-gen 技能负责，但这里也要配合）：

```bash
# 为每个分镜保存提示词
# prompts/01.md, prompts/02.md, ...

# 逐个调用 gemini-image-gen 生成图片
cd <gemini-image-gen 技能目录>

# 第1张
bun scripts/generate-image.ts -p <prompts/01.md 的绝对路径> -o <images/01.png 的绝对路径>

# 等待 15-20 秒（防风控）
sleep 18

# 第2张
bun scripts/generate-image.ts -p <prompts/02.md 的绝对路径> -o <images/02.png 的绝对路径>

# 每生成一张，向用户报告进度
# ✅ 分镜 1/10 已生成（封面）
# ⏳ 等待中...（防风控）
# ⏳ 正在生成分镜 2/10...
```

**每5张额外等待**：每生成5张后，额外等待 30 秒。

**失败重试**：如果某张生成失败，等待 60 秒后重试，最多重试 2 次。如果仍然失败，跳过该分镜并记录，继续生成后续分镜。最后汇总告知用户哪些失败了。

### Step 7: 生成 PDF

所有图片生成完成后，将图片合成为一份 PDF 文件。

使用 Python + reportlab 生成 PDF，每张图片占满一页（16:9 横向布局）：

```python
# 在 ppt-storyboard 技能目录下执行
python scripts/images_to_pdf.py \
  --images-dir <images 目录路径> \
  --output <输出 PDF 文件路径> \
  --title "<PPT标题>"
```

脚本位于本技能的 `scripts/images_to_pdf.py`，功能：
- 按文件名排序读取 `images/` 下所有 PNG 图片
- 每张图片铺满一页（横向 A4 或 16:9 比例）
- 输出为单个 PDF 文件

### Step 8: 返回结果

```
✅ PPT 生成完成！

📁 输出目录：ppt-AI驱动的研发效率提升/
📄 PDF 文件：ppt-AI驱动的研发效率提升/presentation.pdf
📂 分镜提示词：ppt-AI驱动的研发效率提升/prompts/ (10个文件)
🖼️ 分镜图片：ppt-AI驱动的研发效率提升/images/ (10张)

共 10 页，其中 10 页生成成功，0 页失败
```

## 输出目录结构

```
ppt-<名称>/
├── prompts/
│   ├── style-config.md    # 全局样式配置
│   ├── 01.md              # 分镜1 提示词
│   ├── 02.md              # 分镜2 提示词
│   └── ...
├── images/
│   ├── 01.png             # 分镜1 图片
│   ├── 02.png             # 分镜2 图片
│   └── ...
├── storyboard.json        # 完整分镜计划（含叙事策略）
└── presentation.pdf       # 最终 PDF 文件
```

## 依赖

- **gemini-image-gen 技能**：用于生成每个分镜的图片
- **Python 3 + reportlab**：用于将图片合成 PDF（如果没装 reportlab，脚本会自动 pip install）
- **Pillow**：图片处理

## 注意事项

1. 叙事对象不可跳过——不同受众决定了整个演示的基调
2. 分镜计划必须展示给用户并获得确认
3. 样式选择在分镜确认之后，用户可以根据分镜内容再决定样式
4. 图片生成过程中持续报告进度
5. 遇到生成失败不中断整体流程，继续后续分镜
6. PDF 生成是最后一步，确保所有可用图片都已就绪
