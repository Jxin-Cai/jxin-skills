# PDF 视觉样式与转换指南

## 设计理念

**"高端名片式开场"** —— PDF 打开的第一眼就是候选人画像，如同一张精心设计的高端名片。面试官无需翻页即可掌握候选人全貌，后续内容逐章展开证明画像中的每一个标签。

**三层气质**：
1. **沉稳专业**：深炭色底、克制的排版、充裕的留白 → 资深专家的从容
2. **温暖有力**：琥珀橙色点缀、渐变装饰、精致标签 → 对技术的热忱
3. **信息清晰**：层级分明、量化高亮、视觉引导 → 专业的信息架构

## 配色方案

### 琥珀深邃（dark 主题，默认）

灵感来源：Hermès 橙 × 深夜代码编辑器 — 奢华、沉稳、有力量感。

```
画像区背景：  #1C1C1E → #2C2C2E  （深炭渐变，沉稳底色）
姓名文字：    #FFFFFF              （纯白，最高对比度）
画像区正文：  rgba(255,255,255,0.88)（高可读性白）

橙色主调：    #E8740C              （Hermès 琥珀橙 — 品牌色）
暖橙辅助：    #F5A623              （蜂蜜金 — 渐变终点）
橙色光晕：    rgba(232,116,12,0.12)（微光装饰）

标签背景：    rgba(232,116,12,0.10)（极淡橙底）
标签文字：    #E8740C              （橙色）
标签边框：    rgba(232,116,12,0.30)（柔和橙边）

正文区背景：  #FFFFFF              （纯白，清爽干净）
正文文字：    #2D2D2D              （深灰，护眼舒适）
标题文字：    #1C1C1E              （近黑，权威感）
次级文字：    #6B6B6B              （中灰，层级区分）
章节衬底：    #FFF9F3              （暖白，微妙的温度感）
分隔线：      #F0E6DA              （暖灰，不刺眼）
```

### 经典暖橙（classic 主题）

适合投递传统行业或偏保守的公司，画像区为浅色底 + 左侧橙色粗边。

```
画像区背景：  #FFFAF5 → #FFF5EB  （暖白渐变）
姓名文字：    #1C1C1E             （深色）
橙色主调：    #D4661E             （烟熏橙，更沉稳）
```

## 候选人画像区域设计

画像区是 PDF 的"第一屏"，也是决定面试官是否继续阅读的关键。

### 关键设计元素

**左侧橙色边条**（5px）：
- 画像区的视觉锚点，引导视线从左侧开始扫描
- 使用 `border-left: 5px solid #E8740C`

**姓名下方装饰线**（48px 渐变短线）：
- 从 `#E8740C` 渐变到 `#F5A623`
- 体现设计感的同时不喧宾夺主

**右上角光晕**（CSS 伪元素）：
- 半径 160px 的径向渐变，极低透明度
- 增加视觉层次感和高端感

**一句话定位**（tagline）：
- 左侧 3px 橙色竖线
- 背景为极淡橙色 `rgba(232,116,12,0.12)`
- 文字为橙色，14px，略大于正文
- 这是整个画像区视觉权重最高的元素

**技能标签**（skill-tag）：
- 等宽字体（SF Mono / JetBrains Mono）
- 橙色边框 + 极淡橙底
- 圆角药丸形（border-radius: 14px）
- 程序员标识感 + 高端感

## 正文区设计

### 技能矩阵

- 使用“左侧分类 + 右侧技能内容”的单列结构，优先强调对齐感和可扫读性
- 每一行是一个轻量能力块：浅底色、细边框、轻橙色锚点
- 标签宽度固定，内容区自然换行，兼顾信息密度与 A4 稳定排版

### 代表项目

- 使用轻卡片包裹单个项目，增强与普通正文的层级区分
- `背景 / 任务 / 行动 / 结果 / 技术栈` 采用统一小标签分块，保持 STAR 结构清晰
- `结果` 区域额外突出量化成果，但整体仍保持克制专业，不做过度装饰

### 章节标题

- 左侧 4px 橙色渐变条（`linear-gradient(#E8740C, #F5A623)`）
- 16px，700 weight，近黑色
- 使用 `::before` 伪元素实现，不占文档流

### 分隔线

- 中心渐变：`transparent → 橙色 → transparent`
- 从两端淡入淡出，优雅不刺眼
- 高度 1px，margin 18px

### 量化数据高亮

在正文中，加粗的数字可用 `**数字**` 标记，CSS 会自动为 `strong` 内容使用深色加粗样式，让量化成就更醒目。

## 排版规范

### 字体选择

```css

font-family: "PingFang SC", "Noto Sans SC", "Microsoft YaHei",
             -apple-system, "Helvetica Neue", sans-serif;

/* 技能标签 & 代码 — 程序员身份感 */

             "Consolas", monospace;
```

### 字号体系

```
姓名：      32px, weight 800, letter-spacing 2px
一句话定位：14px, weight 500
章节标题：  16px, weight 700
子标题：    14px, weight 600
正文：      12.5px, weight 400, line-height 1.65
标签：      10.5px, weight 500
小字注释：  12px
```

### 页面尺寸

```
纸张：A4 (210mm × 297mm)
页边距：上 12mm, 右 16mm, 下 14mm, 左 16mm
```

## 图表与图片处理

### Mermaid 图表
HTML 模板中嵌入 mermaid.js，图表在浏览器渲染后由 Playwright 截取到 PDF 中。

### 本地图片
自动转换为 base64 内联，确保 PDF 独立可用。

### 远程图片
保持原始 URL，生成 PDF 时需网络连接。

## 使用方法

```bash
# 默认 — 琥珀深邃（深色画像区）
python3 scripts/convert_resume_to_pdf.py resume.md

# 经典暖橙（浅色画像区）
python3 scripts/convert_resume_to_pdf.py resume.md --theme classic

# 仅生成 HTML 预览
python3 scripts/convert_resume_to_pdf.py resume.md --html-only
```

## 自定义样式

如需调整橙色色相或其他配色，修改脚本中 `THEMES` 字典的对应值即可：

```python
THEMES = {
    "dark": {
        "accent": "#E8740C",       # 主橙色
        "accent_warm": "#F5A623",  # 暖橙/渐变终点
        ...
    }
}
```

## PDF 后处理

PDF 生成后，脚本会自动执行以下后处理步骤，确保输出的 PDF 更加专业和安全。

### 元数据写入

默认自动为 PDF 写入以下元数据（可通过 `--no-metadata` 关闭）：

| 字段 | 值 |
|------|------|
| Title | 候选人姓名（从 Markdown H1 标题提取） |
| Author | Resume Craft |
| Subject | Professional Resume |
| Creator | Resume Craft - Expert Resume PDF Generator |

元数据的好处：
- PDF 阅读器标题栏显示候选人姓名而非文件名
- 搜索和归档更方便
- 体现专业度和细节把控

使用 pypdf 实现：`pip install pypdf`

### 密码加密

通过 `--encrypt PASSWORD` 参数可为 PDF 设置密码保护，适用于：
- 包含敏感个人信息（身份证号、手机号）的简历
- 通过公共渠道传输时保护隐私

加密后，打开 PDF 需输入密码。使用 pypdf 的 AES-256 加密。

### 生成验证

通过 `--verify` 参数可将 PDF 首页转为 PNG 预览图，用于：
- 确认渲染效果是否符合预期
- 在不打开 PDF 阅读器的情况下快速检查
- CI/CD 流水线中的自动化验证

预览图保存为 `<filename>.preview.png`，DPI 150，分辨率足以检查排版。

支持两种转换引擎（任选其一安装即可）：
- `pdf2image`：基于 poppler，需要系统安装 poppler-utils
- `pypdfium2`：纯 Python，无系统依赖，推荐

### 三级降级

PDF 渲染引擎按以下顺序自动降级：

```
Playwright (推荐)
  ↓ ImportError / 渲染失败
weasyprint (降级)
  ↓ ImportError / 渲染失败
提示手动打印 (最终方案)
```

- **Playwright**：Chromium 内核，CSS/JS 支持完整，Mermaid 图表可渲染
- **weasyprint**：纯 Python，部分高级 CSS 不支持（如 Mermaid 不会渲染）
- **手动打印**：提供 HTML 文件路径，用户在浏览器中 Ctrl+P 打印
